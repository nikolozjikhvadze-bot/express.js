# express.js
# ⚙️ Express.js REST API Server

ეს არის პროექტის სერვერული ნაწილი (Backend), რომელიც აგებულია **Node.js**-ისა და **Express.js** ფრეიმვორქის ბაზაზე. სერვერი უზრუნველყოფს მონაცემთა ბაზასთან (MongoDB/Mongoose) კავშირს, მომხმარებელთა ავტორიზაციას, უსაფრთხოებას და ამუშავებს ფრონტენდიდან გამოგზავნილ ყველა მოთხოვნას (API Requests).

---

## ✨ ძირითადი ფუნქციონალი (Features)

* **RESTful API:** სუფთა და ორგანიზებული ენდპოინტები (Endpoints) მონაცემების მისაღებად, დასამატებლად, განასახლებლად და წასაშლელად (CRUD ოპერაციები).
* **ავტორიზაცია და უსაფრთხოება:** მომხმარებლის პაროლების ჰეშირება (bcrypt) და უსაფრთხო სესიები/ტოკენები (JWT - JSON Web Tokens).
* **CORS Middleware:** ინტეგრირებული `cors` პაკეტი ფრონტენდ აპლიკაციებთან უსაფრთხო კავშირისთვის.
* **მონაცემთა ვალიდაცია:** სერვერზე შემოსული ინფორმაციის მკაცრი შემოწმება ბაზაში ჩაწერამდე.

---

## 🛠️ ტექნოლოგიური სტეკი

* **Runtime Environment:** Node.js
* **Framework:** Express.js
* **Database ODM:** Mongoose (MongoDB-სთან სამუშაოდ)
* **სხვა პაკეტები:** `dotenv` (გარემო ცვლადებისთვის), `nodemon` (ავტომატური გადატვირთვისთვის დეველოპმენტისას).

---

## 📁 არქიტექტურა (Project Structure)

ბექენდის კოდი სტრუქტურირებულია MVC (Model-View-Controller) პრინციპის მიხედვით:
* **`models/`** - მონაცემთა ბაზის სქემები (მაგ. User, Booking, Trip).
* **`controllers/`** - ძირითადი ბიზნეს ლოგიკა და ფუნქციები.
* **`routes/`** - ენდპოინტების მარშრუტიზაცია (URL სტრუქტურა).
* **`index.js`** / **`server.js`** - სერვერის მთავარი შესავალი ფაილი და კონფიგურაცია.
